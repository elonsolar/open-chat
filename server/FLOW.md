# 数据流程详解

## 完整请求-响应流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        步骤 1: 客户端发送 HTTP 请求                   │
├─────────────────────────────────────────────────────────────────────┤
│  POST http://localhost:3000/v1/chat/completions                     │
│  {                                                                   │
│    "model": "我的客服会话",      ← 插件中的会话名称                   │
│    "messages": [                                                       
│      {"role": "user", "content": "帮我查询北京天气"}                         │
│    ],                                                                 │
│    "stream": false                                                      
│  }                                                                    │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              步骤 2: openai-api-server.js 接收并处理请求             │
├─────────────────────────────────────────────────────────────────────┤
│  1. 验证请求格式                                                       │
│  2. 如果有 tools，转换为文本说明格式：                                   │
│     **get_weather** - 查询天气...                                     │
│  3. 将工具使用说明附加到用户消息                                         │
│     (使用 ```tool_call 代码块格式)                                      │
│  4. 创建 requestData:                                                 │
│     {                                                                 │
│       model: "我的客服会话",                                          │
│       messages: [...],                                                │
│       tools: [...]                                                    │
│     }                                                                 │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                 步骤 3: MessageRouter 发送到 WebSocket               │
├─────────────────────────────────────────────────────────────────────┤
│  1. 生成唯一的 requestId (UUID)                                       │
│  2. 创建 Promise 并设置 30 秒超时                                      │
│  3. 将 Promise 存储在 pendingRequests Map 中                           │
│  4. 发送 WebSocket 消息到所有连接的插件：                                │
│                                                                      │
│     {                                                                │
│       "type": "chat_request",                                        │
│       "requestId": "uuid-1234-5678",                                 │
│       "model": "我的客服会话",                                        │

│     }                                                                │

                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│          步骤 4: 插件 background.js WebSocketManager 接收            │
├─────────────────────────────────────────────────────────────────────┤
│  WebSocketManager.handleMessage():                                   │
│  识别 type === "chat_request"                                        │
│  调用 handleChatRequest(message)                                     │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              步骤 5: 通过会话名称查找并使用会话                       │
├─────────────────────────────────────────────────────────────────────┤
│  handleChatRequest():                                               │
│  1. 提取用户消息                                                      │
│  2. 使用 model 参数（会话名称）查找会话：                               │

│                                                                      │

│              步骤 7: 触发角色发送消息到 AI 平台                      │
├─────────────────────────────────────────────────────────────────────┤
│  AIMessageManager 依次处理每个角色:                                   │
│                                                                      │

│    ├─ 填充用户消息到输入框                                            │
│    ├─ DeepSeekAdapter.submitMessage()                              │
│    └─ 等待响应...                                                   │
│                                                                      │
│  [等待 AI 响应]                                                      │
│    └─ MutationObserver 检测到新消息                                  │

│    └─ 提取 AI 响应内容                                               │
│    └─ Content Script 发送 aiResponse 消息回 background              │

│  收集到的响应:                                                        │

## 关键数据结构

### WebSocket 请求 (服务器 → 插件)
```json
{
  "type": "chat_request",
  "requestId": "uuid-1234-5678",
  "model": "我的客服会话",
  "messages": [
    {
      "role": "user",
      "content": "帮我查询北京天气\n\n你可以使用以下工具：\n..."
    }
  ],
  "tools": [...],
  "timestamp": 1234567890
}
```

### WebSocket 响应 (插件 → 服务器)
```json
{
  "type": "ai_response",
  "requestId": "uuid-1234-5678",
  "content": "我来帮您查询北京天气...",
  "conversation_id": "conv-123",
  "conversation_name": "我的客服会话",
  "timestamp": 1234567890
}
```

### 会话结构
```json
{
  "id": "conv-123",
  "name": "我的客服会话",
  "roleIds": ["role-001", "role-002"],
  "contextMode": "self",
  "messages": [
    {
      "id": "msg-001",
      "isUser": true,
      "content": "帮我查询北京天气",
      "timestamp": 1234567890
    },
    {
      "id": "msg-002",
      "isUser": false,
      "roleId": "role-001",
      "content": "我来帮您查询...",
      "provider": "deepseek",
      "timestamp": 1234567895
    }
  ]
}
```

## 时间线

| 时间 | 事件 | 说明 |
|------|------|------|
| T+0ms | 客户端发送 HTTP 请求 | 开始 |
| T+5ms | 服务器接收并处理 | 验证、格式转换 |
| T+10ms | 发送到 WebSocket | 转发到插件 |
| T+15ms | 插件接收请求 | background.js 处理 |
| T+20ms | 查找会话 | 通过名称查找 |
| T+30ms | 添加消息到会话 | 保存到 storage |
| T+50ms | 发送到 AI 平台 | 打开/激活标签页 |
| T+100ms | 填充并发送消息 | content script 操作 |
| T+5000ms | AI 响应返回 | 等待 AI 生成 |
| T+5100ms | 检测并提取响应 | MutationObserver |
| T+5200ms | 添加到会话 | 保存响应 |
| T+5300ms | 发回服务器 | WebSocket 响应 |
| T+5310ms | 返回给客户端 | HTTP 响应 |
| T+5310ms | 完成 | 总耗时约 5.3 秒 |

## 错误处理

### 会话不存在
```json
{
  "error": {
    "message": "会话不存在: 我的客服会话。请先在插件中创建名为 \"我的客服会话\" 的会话。",
    "type": "invalid_request_error"
  }
}
```

### 会话无角色
```json
{
  "error": {
    "message": "会话 \"我的客服会话\" 没有配置角色。请在插件中为该会话添加角色。",
    "type": "invalid_request_error"
  }
}
```

### 超时
```json
{
  "error": {
    "message": "请求超时（30秒）",
    "type": "timeout_error"
  }
}
```
